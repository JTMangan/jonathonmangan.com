The code above for the Arduino is some test code for how the Arduino makes the car move. It starts with the #include statement which allows you to implement different databases into your code to use specific commands. Within mine I specifically have the Adafruit_MotorShield which allows us to implement motors. This is shown when I make the AFMS which is the motor shield. This motor shield allows you to initialize and run motors. This is first done by setting the motors through the command Adafruit_DCMotor *motorR1 = AFMS.getMotor(1). This sets the name of the motor(motorR1), and AFMS.getMotor() tells you which port of the shield by placing the number in the parentheses. Further down in the code you set speed and the direction, setting speed is done with "motorName->setSpeed(integer between 0 - 255)" From there to do direction you either right forward or backward, with forward keeping all the motors moving in a positive direction then if backward to be in a negative direction causing the wheels to change.
Over the last month I've been trying to send info between a Raspberry pi and an Arduino. What I've learned about is the serial port on the Arduino. This issue within the serial port is that it only excepts one piece of data at a time so as the next step I'm gonna make a key within the data sent so that it, does different movements based on my key whether it's forward, backward, left or right.
